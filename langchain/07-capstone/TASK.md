# Task 7: Capstone — RAG Application

## Objective
Build a retrieval-augmented generation application over a custom document corpus with a FastAPI backend.

## What to Learn
- End-to-end RAG architecture: ingest, embed, store, retrieve, generate
- Combining LangChain with a web framework
- Production concerns: chunking strategy, retrieval quality, prompt engineering
- Evaluation of RAG output quality

## Project: DevOps Knowledge Base

Build a RAG application that answers questions about DevOps/infrastructure topics using a custom document corpus.

### Requirements

1. **Document Corpus** — Create or collect 10+ text/markdown documents covering topics you know (Kubernetes, Docker, CI/CD, networking, cloud services). Store in `corpus/`.

2. **Ingestion Pipeline** — Script (`ingest.py`) that:
   - Loads all documents from `corpus/`
   - Splits them with appropriate chunk size/overlap
   - Embeds and stores in a persistent vector store (Chroma with persistence or FAISS saved to disk)

3. **RAG Chain** — Module (`rag.py`) that:
   - Loads the vector store
   - Builds a RAG chain with a well-crafted prompt
   - Supports source attribution (return which documents were used)
   - Handles "I don't know" when context is insufficient

4. **FastAPI Backend** — API (`api.py`) with:
   - `POST /ask` — takes `{"question": "..."}`, returns `{"answer": "...", "sources": [...]}`
   - `POST /ingest` — triggers re-ingestion
   - `GET /health` — health check

5. **Evaluation** — Script (`evaluate.py`) that:
   - Defines 10+ test questions with expected answers
   - Runs them through the RAG chain
   - Scores using at least one custom evaluator
   - Prints a summary report

### Structure
```
07-capstone/
├── api.py
├── rag.py
├── ingest.py
├── evaluate.py
├── corpus/
│   ├── kubernetes-pods.md
│   ├── docker-networking.md
│   └── ...
├── vectorstore/       # generated by ingest.py

```

### Evaluation Criteria
- Clean separation: ingestion, retrieval, API are independent modules
- RAG chain correctly attributes sources
- Handles out-of-scope questions gracefully
- Evaluation script runs and produces meaningful scores
- FastAPI endpoints work with curl/httpie
