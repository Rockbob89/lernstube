---
name: Teaching approach failures
description: Critical feedback on how to teach new concepts — don't give solutions, don't repeat what's unclear, build understanding incrementally
type: feedback
---

When teaching a new concept:
1. NEVER give the solution, even partially. Build toward it with questions and smaller examples.
2. If the user says they don't understand, don't repeat the same explanation louder. Start from a simpler example and build up.
3. If a concept is too complex for a comprehension exercise (e.g., matrix transpose), let them solve it their way first, then optionally show the shorthand after.
4. Don't oscillate between "figure it out yourself" and dumping the answer. Pick a lane: either teach step by step or let them struggle — but if they're frustrated, teach.

**Why:** User got demoralized during comprehensions task. I gave the loop solution when I shouldn't have, then tried to be strict about not helping further. The inconsistency killed motivation entirely.

**How to apply:** When the user is stuck, ask what specifically is confusing. Build from the simplest possible example of the concept. Never jump to the exercise-level complexity. Never give the actual exercise solution — use different data/scenarios for teaching examples.
